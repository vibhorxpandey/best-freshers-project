# 100 First-Year Projects for Freshers

A curated list of 100 project ideas for first year CS/engineering students — split into **Easy**, **Medium**, and **Hard** tiers so you can level up deliberately instead of guessing what to build next line i did in starting.

## How to use this repo

- **New to coding?** Start in [`EASY.md`](./EASY.md). Each of these is buildable in a weekend and teaches one core skill at a time.
- **Comfortable with one language + basic APIs/DBs?** Move to [`MEDIUM.md`](./MEDIUM.md). These involve real CRUD apps, persistence, and third-party integrations.
- **Shipped a few projects already?** Try [`HARD.md`](./HARD.md). These push into systems programming, distributed concepts, and deeper ML/AI — the kind of projects that make strong resume/portfolio pieces even in year one.
- Pick things outside your comfort zone on purpose. Breadth in year one pays off later.
- Each entry has a suggested stack — treat it as a starting point, not a rule. Swap languages/frameworks freely.

## Structure

| Tier | Count | Focus |
|---|---|---|
| [Easy](./EASY.md) | 34 | Fundamentals, scripting, small UI/CLI tools |
| [Medium](./MEDIUM.md) | 33 | Full CRUD apps, APIs, basic ML, real integrations |
| [Hard](./HARD.md) | 33 | Systems programming, distributed concepts, deeper ML/AI, from-scratch builds |

**Total: 100 projects**

## Suggested workflow per project

1. Write a 3-line spec before touching code (what it does, inputs/outputs, done-criteria).
2. Build the ugliest version that works first.
3. Push to GitHub with a real README — this is what recruiters and reviewers actually look at.
4. Only then go back and polish/refactor.
